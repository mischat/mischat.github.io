Capturing your Digital Persona at Source : Lifelogging

  • More and more transaction/social interactions are happening online
  • This has been labelled 'The Disappearence of the Body'
  • Our daily lives leave behind more evidence that we can collect and curate, and the indiscriminating collection of such evidence is what has been coined Lifelogging
  • Lifelogging can either be :
    • Passive : storing by-productsof the life one would have lived anyways
    • Active : one surrounds oneself with sensors and information capture tools to create as rich a picture of one’s life as possible.

Uniform Resource Identifiers (URI)

Who knows what a URI is?

  • A URI is a compact sequence of characters that identifies an abstract concept or a physical resource - RFC3986
  • foo://bar.com:8001/boo/x?something=value#fragment
  • \___/\___________/\____/ \_____________/ \______/
  • scheme authority path query fragment
  • URIs are NOT merely URLs!
  • And I should stress that cool ones don't change
    • http://
    • mailto:
    • tag:
    • file://
    • isbn:
    • urn://
    • ...

RDF isn't that bad ...

  • RDF is not simply a flavour of XML, but it can be serialised in XML, but it is very ugly 
    <?xml version="1.0" encoding="utf-8"?>
<rdf:RDF
	xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#"
	xmlns:foaf="http://xmlns.com/foaf/0.1/">
 <foaf:PersonalProfileDocument rdf:about="">
	<foaf:primaryTopic rdf:resource="http://mmt.me.uk/foaf.rdf#mischa"/>
 </foaf:PersonalProfileDocument>

 <foaf:Person rdf:ID="mischa">
	<foaf:homepage rdf:resource="http://mmt.me.uk/"/>
 </foaf:Person>
</rdf:RDF>
  • It can be serialised in other ways to make it easier to read ... 
     @prefix rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
 @prefix foaf: <http://xmlns.com/foaf/0.1/> .
  <> a foaf:PersonalProfileDocument ;
  	 foaf:primaryTopic <#mischa> .
  <#mischa> a foaf:Person ;
	foaf:homepage <http://mmt.me.uk/> .
  • It can even more verbose: 
        <http://mmt.me.uk/foaf.rdf> <http://www.w3.org/1999/02/22-rdf-syntax-ns#type> <http://xmlns.com/foaf/0.1/PersonalProfileDocument> .

So what does all of this mean?

  • Represent your data in triples, use unique identifiers, and make use of as many shared conceptualisations as you can
  • So what are triples?: Subject Predicate Object
  • Implied semantics in column name in a class db
  • And this is very similar in form to the simplest of English sentences
  • some guy called X has a homepage X
  •  <#personX> foaf:homepage <http://foo.com/bar> .
  • Need to be able to bung any Class/property URI into a web browser to get its description

FOAF : The Friend-Of-A-Friend Ontology

  • 'The Friend of a Friend (FOAF) project began early in 2000 as an "experimental linked information project", created by Dan Brickley and Libby Miller.'
  • RDF : http://xmlns.com/foaf/0.1/
  • HTML : http://www.foaf-project.org/
  • FOAF makes it possible for people to build large scale, decentralised, open social networks that are controlled by individual users.
  • Sites such as : LiveJournal, hi5, vox, friendfeed?, google???,...
  • Reflectors : twitter, flickr, lastfm, delicious, ...

OpenID Social Verifier (1): http://foaf.qdos.com/verify-about

  • The service is implemented as a RESTful HTTP service.
  • You send requests to the service with an identifier for you and the OpenID you wish to verify.
  • If the OpenID is within 2 people of you then the service returns YES, and the distance between you and them, 0 being an ID for you, and 2 being 2 people away, for example: 
      <verify>
  <result degree="2">YES<result>
  <verify>
  • The endpoint of the REST service is http://foaf.qdos.com/verify and the arguments are action, qdosuri, ifp, ifpval and openid.
  • action Check or List
