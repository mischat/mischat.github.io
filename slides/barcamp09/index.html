Uniform Resource Identifiers (URI)

Who knows what a URI is?

  • A URI is a compact sequence of characters that identifies an abstract concept or a physical resource - RFC3986
  • foo://bar.com:8001/boo/x?something=value#fragment
  • \___/\___________/\____/ \_____________/ \______/
  • scheme authority path query fragment
  • URIs are NOT merely URLs!
  • And I should stress that cool ones don't change
    • http://
    • mailto:
    • tag:
    • file://
    • isbn:
    • urn://
    • ...

RDF isn't that bad ...

  • RDF is not simply a flavour of XML, but it can be serialised in XML, but it is very ugly 
    <?xml version="1.0" encoding="utf-8"?>
<rdf:RDF
	xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#"
	xmlns:foaf="http://xmlns.com/foaf/0.1/">
 <foaf:PersonalProfileDocument rdf:about="">
	<foaf:primaryTopic rdf:resource="http://mmt.me.uk/foaf.rdf#mischa"/>
 </foaf:PersonalProfileDocument>

 <foaf:Person rdf:ID="mischa">
	<foaf:homepage rdf:resource="http://mmt.me.uk/"/>
 </foaf:Person>
</rdf:RDF>
  • It can be serialised in other ways to make it easier to read ... 
     @prefix rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
 @prefix foaf: <http://xmlns.com/foaf/0.1/> .
  <> a foaf:PersonalProfileDocument ;
  	 foaf:primaryTopic <#mischa> .
  <#mischa> a foaf:Person ;
	foaf:homepage <http://mmt.me.uk/> .
  • It can even more verbose: 
        <http://mmt.me.uk/foaf.rdf> <http://www.w3.org/1999/02/22-rdf-syntax-ns#type> <http://xmlns.com/foaf/0.1/PersonalProfileDocument> .
    <http://mmt.me.uk/foaf.rdf> <http://xmlns.com/foaf/0.1/primaryTopic> <http://mmt.me.uk/foaf.rdf#mischa> .
    <http://mmt.me.uk/foaf.rdf#mischa> <http://www.w3.org/1999/02/22-rdf-syntax-ns#type> <http://xmlns.com/foaf/0.1/Person> .
    <http://mmt.me.uk/foaf.rdf#mischa> <http://xmlns.com/foaf/0.1/homepage> <http://mmt.me.uk/> .

So what does all of this mean?

  • Represent your data in triples, use unique identifiers, and make use of as many shared conceptualisations as you can
  • So what are triples?: Subject Predicate Object
  • Implied semantics in column name in a class db
  • And this is very similar in form to the simplest of English sentences
  • some guy called X has a homepage X
  •  <#personX> foaf:homepage <http://foo.com/bar> .
  • Need to be able to bung any Class/property URI into a web browser to get its description

Lifelogging & Empowerment

  • Don't trust people to store your data on your behalf
  • Capture your own context and your own content
  • Store your own data, give yourself a WebID, and use that as the subject whenever you want to create data about yourself
  • Garlik FOI request report
  • Store your own data and use pushback technology like mhausenblas's RDF -> API RDF Pushback work
  • Lifelogging: Privacy and Empowerment with Memories for Life
  • Rutgers Uni., Hasan Elahi, who was arrested in 2002 on serious terrorism allegations despite being absolutely innocent of any criminal or terrorist activity, has taken to lifelogging and posting the information on the Web as a pre-emptive alibi, see: http://trackingtransience.net/

How would you do this?

  • FOAF : Documents on the web describing people and how their friends are
  • Public / Private : via OAuth / FOAF+SSL
  • An index of FOAF files, to give you links in, otherwise you would only have links out ....
  • Associate the Contextual Information, with the Content, that it to your WebID/FOAF Person via RDF
  • Using Event Ontology, geoLocation Ontology, soic ontology, and so on ...
